# WebEnvoy App Human Workbench Story

## User Story

- Schema marker: `loom-user-story/v1`
- Story statement: 作为使用 WebEnvoy 管理网站工作的市场、运营或产品人员，我希望在一个面向人的 App 中管理账号与浏览器、使用站点技能、创建或观察所有入口产生的任务并直接消费业务结果，从而无需理解内部运行系统也能完成工作、处理异常并决定下一步。
- Actor: 使用 WebEnvoy 管理网站账号、浏览器环境和网页任务的市场、运营、产品等人类业务用户。
- Capability: 通过 App 的 Work、Browser、Library 三个业务域创建和观察任务、管理账号与浏览器实例、使用站点技能，并在异常或权限边界出现时继续、接管或终止工作。
- Outcome: 用户无需理解 Core、Harbor、Lode、refs 或 runtime 状态，就能看清可执行动作、业务结果、失败原因和下一步；需要排查时再展开来源、运行详情和诊断。
- Business value: App 成为 WebEnvoy 面向人类的统一工作台，让 App、CLI、MCP、API、SDK 和 Agent 产生的任务共享可理解、可恢复、可消费的桌面体验。
- Out of scope:
  - 不把 App 变成 Agent 聊天容器、任意网页自动化入口或 owner truth 存储。
  - 不把小红书当成产品边界；它只承担当前 live validation slice。
  - BOSS 生产任务保持 deferred，不展示为可运行。
  - 不在本 story 决定生产组件、像素、最终路由、Core/Harbor/Lode 接口实现或真实写入能力。
  - 不把运行 refs、内部 evidence 或日志作为默认业务内容。

## Acceptance Scenarios

所有验收场景都使用业务可读的 Given / When / Then，描述可观察的产品行为，
不描述实现步骤或测试脚本。

<a id="scenario-s1"></a>
### Scenario S1: 从账号身份开始工作

- Scenario id: `S1`
- Scenario locator: `docs/stories/APP-308.md#scenario-s1`
- Dimension: `happy_path`

**Given**

用户拥有一个可用或需要登录的账号身份。

**When**

用户在 Browser 中选择该账号身份。

**Then**

- App 展示该账号当前是否可用以及“打开浏览器”或“选择技能”的主要动作。
- 需要登录时，App 直接提供登录入口和完成后的继续路径。
- 手动浏览不会被表述为自动任务或业务结果。

<a id="scenario-s2"></a>
### Scenario S2: 从站点技能开始工作

- Scenario id: `S2`
- Scenario locator: `docs/stories/APP-308.md#scenario-s2`
- Dimension: `alternative_path`

**Given**

用户在 Library 中找到一个满足业务目标的站点技能。

**When**

用户查看技能用途、所需输入和预期结果，并点击“去使用”。

**Then**

- App 打开已预选该站点技能的任务创建页，并只列出兼容账号身份。
- 任务输入表单与该技能声明的输入字段、类型、选项和约束一致。
- 任务创建前，用户可以看清将读取、准备、提交或改变什么。
- 技能包、内部测试和维护信息不会抢占主要流程。

<a id="scenario-s3"></a>
### Scenario S3: 从 Work 创建任务

- Scenario id: `S3`
- Scenario locator: `docs/stories/APP-308.md#scenario-s3`
- Dimension: `happy_path`

**Given**

用户已经进入 Work，并知道希望完成的业务目标。

**When**

用户新建任务并选择站点技能和账号身份。

**Then**

- App 根据所选站点技能展示对应输入表单，而不是开放式通用输入框。
- App 在提交前校验必填项、字段类型、选项和约束，并展示任务目标与当前授权范围。
- 创建后，任务进入统一的运行、结果和异常恢复体验。
- 若缺少技能、兼容账号或可用浏览器，App 展示可执行的补救路径而不是创建不可运行任务。

<a id="scenario-s4"></a>
### Scenario S4: 观察其他入口创建的任务

- Scenario id: `S4`
- Scenario locator: `docs/stories/APP-308.md#scenario-s4`
- Dimension: `alternative_path`

**Given**

CLI、MCP、API、SDK 或 Agent 已经创建一个 WebEnvoy 任务。

**When**

用户打开 Work 并查看该任务。

**Then**

- 该任务与 App 创建的任务使用一致的业务状态、结果和恢复体验。
- App 标识任务来源，但不会按技术入口拆成不同任务产品。

<a id="scenario-s5"></a>
### Scenario S5: 查看任务结果摘要

- Scenario id: `S5`
- Scenario locator: `docs/stories/APP-308.md#scenario-s5`
- Dimension: `happy_path`

**Given**

一个任务已经结束，或已经产生可以消费的阶段性业务结果。

**When**

用户打开该任务。

**Then**

- App 首先展示任务目标、完成状态、结果摘要、结果数量或写操作状态以及可执行下一步。
- 部分完成、结果未知和仍在处理中使用不同业务状态，不会统一显示为成功。
- 来源摘要和原页面入口位于次级层级，运行详情与诊断默认折叠。
- 未知结果类型仍有可读的通用详情，而不是把原始数据作为首选结果。

<a id="scenario-s6"></a>
### Scenario S6: 空结果或任务失败

- Scenario id: `S6`
- Scenario locator: `docs/stories/APP-308.md#scenario-s6`
- Dimension: `negative_path`

**Given**

任务返回空结果、输入错误、技能失效、登录失效、访问限制或未知结果。

**When**

用户查看该任务。

**Then**

- App 先说明业务影响和可执行下一步，例如修改输入、重新登录、重试、打开现场或终止。
- 技术错误和内部运行记录只在用户主动打开诊断时出现。
- 不存在真实结果时，App 不会用样例或历史结果伪装成功。

<a id="scenario-s7"></a>
### Scenario S7: Provider 不可用并恢复

- Scenario id: `S7`
- Scenario locator: `docs/stories/APP-308.md#scenario-s7`
- Dimension: `environment_interruption`

**Given**

账号或任务所需的浏览器 provider 未安装、损坏、版本不兼容或无法启动。

**When**

用户从 Browser 或原任务选择修复问题。

**Then**

- App 根据 provider 来源提供检测、下载、安装、更新、修复、重检或打开外部管理入口。
- 安装或更新前，App 展示受影响的账号身份；完成后必须通过版本、完整性和启动验证才恢复为可用。
- 恢复成功后返回原账号身份或任务上下文；失败时保留清楚、可执行的恢复选择。

<a id="scenario-s8"></a>
### Scenario S8: 需要人工介入

- Scenario id: `S8`
- Scenario locator: `docs/stories/APP-308.md#scenario-s8`
- Dimension: `environment_interruption`

**Given**

任务因登录、验证码、访问限制、文件选择或未知页面状态需要人工操作。

**When**

App 通知用户，用户选择打开对应浏览器现场，完成操作后点击“已完成，继续”或反馈“无法完成”。

**Then**

- App 在收到“已完成，继续”后校验页面、登录或资源状态是否满足恢复条件。
- 校验成功时，系统结束人工接管并继续完成原任务。
- 校验失败时，App 说明仍缺少什么，并允许再次接管、重试校验或终止；不会直接把人工介入报为任务成功。

<a id="scenario-s9"></a>
### Scenario S9: 授权范围不足

- Scenario id: `S9`
- Scenario locator: `docs/stories/APP-308.md#scenario-s9`
- Dimension: `security_permission`

**Given**

任务或浏览器操作属于已声明能力，但按全局、站点技能和任务配置计算后仍未获得授权。

**When**

App 请求用户决定是否允许该动作。

**Then**

- App 清楚展示目标和动作，并提供允许一次、当前任务允许或拒绝。
- 超出已声明能力、目标或有效期的动作直接拒绝，不提供扩大能力声明的入口。
- 所有任务入口使用同一授权结果，不为单个站点建立独立审批流程。

<a id="scenario-s10"></a>
### Scenario S10: 管理浏览器实例生命周期

- Scenario id: `S10`
- Scenario locator: `docs/stories/APP-308.md#scenario-s10`
- Dimension: `edge_case`

**Given**

用户已经选择一个账号身份，且该身份拥有空闲、运行中或需要接管的浏览器实例。

**When**

用户查看该账号的浏览器实例。

**Then**

- App 根据当前状态提供启动、查看、接管、释放或停止等有效动作。
- 用户可以看清当前使用者、站点和是否需要处理，但内部实例标识默认隐藏。
- 同一账号身份已有活动实例时，“打开”复用该实例并提供查看或接管，不静默启动第二个争用同一环境的实例。
- 停止或释放后，账号状态及时更新，不继续显示为正在使用。

<a id="scenario-s11"></a>
### Scenario S11: 按需查看来源与诊断

- Scenario id: `S11`
- Scenario locator: `docs/stories/APP-308.md#scenario-s11`
- Dimension: `edge_case`

**Given**

用户或维护者需要核对结果来源或排查失败。

**When**

用户主动展开来源、运行详情或诊断。

**Then**

- App 展示来源页面、时间、业务阶段以及可供排查的技术记录。
- 关闭详情后，主要业务结果、失败原因和下一步仍保持清晰，不被诊断信息替代。

<a id="scenario-s12"></a>
### Scenario S12: 创建账号身份

- Scenario id: `S12`
- Scenario locator: `docs/stories/APP-308.md#scenario-s12`
- Dimension: `happy_path`

**Given**

用户没有适合目标站点或任务的账号身份。

**When**

用户在 Browser 或任务创建页选择“创建账号身份”，并填写名称、站点、账号标识，选择浏览器 provider 和环境预设。

**Then**

- App 创建以账号为中心的身份环境，并将它加入 Browser 和兼容任务的账号选择列表。
- 创建表单只展示所选 provider 支持的环境配置；代理、地区、语言、时区和指纹设置可以使用推荐值或进入高级设置。
- 需要登录时，App 提供打开浏览器完成登录并返回的连续路径。
- 密码、Cookie、令牌和浏览器档案内容不会作为普通表单数据展示或复制。
- 创建完成后，App 清楚展示该身份是否可用、需要登录或需要修复。

<a id="scenario-s13"></a>
### Scenario S13: 管理站点技能生命周期

- Scenario id: `S13`
- Scenario locator: `docs/stories/APP-308.md#scenario-s13`
- Dimension: `negative_path`

**Given**

一个站点技能存在可用、需要更新、可能失效或需要修复的状态。

**When**

用户在 Library 中查看或管理该技能。

**Then**

- App 用业务语言说明技能用途、当前可用性、版本变化和受影响任务。
- 普通用户获得更新、停用或选择替代技能等有效动作；维护者可以进入测试和修复草稿。
- 未通过验证的修复或样例不会被展示为已恢复的生产能力。

<a id="scenario-s14"></a>
### Scenario S14: 配置四级授权范围

- Scenario id: `S14`
- Scenario locator: `docs/stories/APP-308.md#scenario-s14`
- Dimension: `security_permission`

**Given**

用户希望为不同入口产生的任务和浏览器操作配置一致且可覆盖的授权范围。

**When**

用户分别在 Settings、站点技能详情、任务详情或单次确认中设置授权。

**Then**

- App 提供全局、站点技能、任务和单次四个授权级别。
- 站点技能继承或覆盖全局默认；任务继承或覆盖站点技能；单次决定只覆盖当前动作。
- 当前动作的有效授权按单次、任务、站点技能、全局的顺序采用最具体且有效的配置。
- 浏览器环境操作不强行套用站点技能级别；它使用全局、关联任务或当前操作、单次授权。
- 更具体的授权只在已声明能力和目标范围内生效，不能扩大站点技能本身未声明的动作。
- App 清楚展示当前有效授权来自哪个级别；单次决定不会静默修改任务、技能或全局配置。

<a id="scenario-s15"></a>
### Scenario S15: 按站点和业务场景浏览技能

- Scenario id: `S15`
- Scenario locator: `docs/stories/APP-308.md#scenario-s15`
- Dimension: `happy_path`

**Given**

Library 中存在来自多个站点、服务不同业务场景的站点技能。

**When**

用户打开 Library，选择站点分组或业务场景标签进行筛选。

**Then**

- Library 首先按站点组织技能，并在每个技能上显示所属站点。
- 技能使用数据采集、内容发布、内容下载、内容浏览等业务场景标签。
- 用户可以按一个或多个标签筛选，并与站点分组或搜索条件共同使用。
- 没有匹配技能时，App 展示当前筛选条件和清除筛选入口，而不是空白页面。

<a id="scenario-s16"></a>
### Scenario S16: 按站点技能要求填写任务

- Scenario id: `S16`
- Scenario locator: `docs/stories/APP-308.md#scenario-s16`
- Dimension: `negative_path`

**Given**

用户选择的站点技能声明了必填项、可选项、字段类型、候选值、默认值和校验约束。

**When**

App 展示任务创建表单，用户填写并尝试创建任务。

**Then**

- 表单只呈现该技能声明的业务输入，不提供站点技能要求之外的开放式任务输入。
- 字段标签、说明、控件、默认值和候选项与技能声明一致。
- 输入不满足必填、类型或约束时，App 在对应字段说明问题并阻止创建。
- 输入有效时，提交的任务输入与用户看到并确认的字段一致。

### Scenario S21: 查找和组织账号身份

<a id="scenario-s22"></a>
### Scenario S22: 理解身份、环境和实例关系

- Scenario id: `S22`
- Scenario locator: `docs/stories/APP-308.md#scenario-s22`
- Dimension: `edge_case`

**Given**

用户查看一个账号身份及其浏览器运行状态。

**When**

用户打开账号身份详情或尝试再次启动它。

**Then**

- App 将账号身份作为主要对象；一个账号身份对应一个持久浏览器环境，运行实例是该环境的一次临时运行。
- 同一环境同一时间最多拥有一个活动可控实例；再次打开时复用现有实例并提供查看、接管或停止。
- 需要独立环境时，用户创建或复制为新的账号身份，而不是让两个实例同时修改同一环境。

<a id="scenario-s23"></a>
### Scenario S23: 导入已有账号环境

- Scenario id: `S23`
- Scenario locator: `docs/stories/APP-308.md#scenario-s23`
- Dimension: `alternative_path`

**Given**

用户已有受支持的 WebEnvoy 环境、系统浏览器档案或 provider 管理的环境。

**When**

用户在 Browser 中选择“导入账号身份”并选择来源。

- 运行详情与诊断默认折叠；定位或预览回合不会改变线程、复制运行事实或丢失其他回合结果。
