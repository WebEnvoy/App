# APP-236

## Static Facts

- Item ID: APP-236
- Goal: Let a user explicitly confirm completed manual authentication for an active, user-controlled Harbor managed session and render only Harbor's returned public identity state.
- Scope: App Harbor client, identity recovery UI, focused smoke coverage, and APP-236-specific Loom evidence only.
- Excluded: Harbor route implementation, App #265 shared carriers and runtime supervisor, Core/Lode changes, automatic login inference, DOM/Cookie/storage inspection, credentials, verification codes, raw profile/evidence, task execution, and external writes.

## Host Binding

- Anchor Work Item: https://github.com/WebEnvoy/App/issues/236
- Harbor dependency: https://github.com/WebEnvoy/Harbor/issues/241
- Branch: work/app-236-manual-auth-sync

## Safety Boundary

- App sends a bodyless user-confirmation intent only to `POST /runtime/sessions/{runtime_session_ref}/manual-authentication-completed`.
- The action is fail-closed unless the projection is a Harbor live identity with an active user-held managed session.
- App only displays the returned public identity state and then refetches owner projections; it never infers authentication from browser/page signals or stores sensitive material.
