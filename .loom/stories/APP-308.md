# WebEnvoy App Human Workbench Story

## User Story

- Schema marker: `loom-user-story/v1`
- Actor: 使用 WebEnvoy 管理网站账号、浏览器环境和网页任务的市场、运营、产品等人类业务用户。
- Capability: 通过 App 的 Work、Browser、Library 三个业务域创建和观察任务、管理账号与浏览器实例、使用站点技能，并在异常或权限边界出现时继续、接管或终止工作。
- Outcome: 用户无需理解 Core、Harbor、Lode、refs 或 runtime 状态，就能看清可执行动作、业务结果、失败原因和下一步；需要排查时再展开来源、运行详情和诊断。
- Business value: App 成为 WebEnvoy 面向人类的统一工作台，让 App、CLI、MCP、API、SDK 和 Agent 产生的任务共享可理解、可恢复、可消费的桌面体验。
- Out of scope:
  - 不把 App 变成 Agent 聊天容器、任意网页自动化入口或 owner truth 存储。
  - 不把小红书当成产品边界；它只承担当前 live validation slice。
  - BOSS 生产任务保持 deferred，不展示为可运行。
  - 不在本 story 决定生产组件、像素、最终路由、Core/Harbor/Lode 接口实现或真实写入能力。
  - 不把运行 refs、内部 evidence 或日志作为默认业务内容。

## Acceptance Scenarios

### Scenario S1: 从账号身份开始工作

- Scenario id: `S1`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s1`
- Dimension: `happy_path`

Given
- 用户拥有一个可用或需要登录的账号身份。

When
- 用户在 Browser 中选择该账号身份并打开浏览器，或选择兼容站点技能创建任务。

Then
- App 清楚展示当前实例、登录或任务状态以及下一步主动作。
- 手动浏览实例不会被表述为 Core Task、Run 或业务结果。

### Scenario S2: 从站点技能开始工作

- Scenario id: `S2`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s2`
- Dimension: `alternative_path`

Given
- 用户在 Library 中找到一个满足业务目标的站点技能。

When
- 用户查看其业务用途、输入输出和动作要求并选择使用。

Then
- App 只要求选择兼容账号身份并填写必要业务输入。
- 技能的 schema、package refs 和内部测试信息不会抢占主要流程。

### Scenario S3: 观察其他入口创建的任务

- Scenario id: `S3`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s3`
- Dimension: `alternative_path`

Given
- CLI、MCP、API、SDK 或 Agent 已通过公共任务入口创建任务。

When
- 用户打开 Work。

Then
- 该任务与 App 创建的任务使用一致的业务状态、结果和恢复体验。
- App 标识任务来源，但不按技术入口割裂任务模型。

### Scenario S4: 消费业务结果

- Scenario id: `S4`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s4`
- Dimension: `happy_path`

Given
- 一个任务已返回内容、记录集合、媒体、指标、账号资料或写操作状态。

When
- 用户打开任务结果。

Then
- App 首先用适合结果类型的正文、列表、表格或状态展示业务结果。
- 来源摘要和原页面入口位于次级层级，运行详情与诊断默认折叠。
- 未知结果类型仍有可读的通用键值或表格展示，而不是以 raw JSON 为首选。

### Scenario S5: 空结果或任务失败

- Scenario id: `S5`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s5`
- Dimension: `negative_path`

Given
- 任务返回空结果、输入错误、技能失效、登录失效、访问限制或未知结果。

When
- 用户查看任务。

Then
- App 先说明业务影响和可执行下一步，例如修改输入、重新登录、重试、打开现场或终止。
- 原始错误、endpoint、trace 和 refs 只在用户主动打开诊断时出现。
- 不存在真实 runtime 结果时，不用 fixture 或历史结果伪装成功。

### Scenario S6: Provider 不可用并恢复

- Scenario id: `S6`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s6`
- Dimension: `environment_interruption`

Given
- 任务所需 provider 未安装、损坏、版本不兼容或无法启动。

When
- 用户从 Browser 或原任务进入恢复流程。

Then
- App 根据 provider 来源提供检测、下载、安装、更新、修复、重检或打开外部管理入口。
- 恢复成功后返回原账号身份或任务上下文；失败时保留清楚、可执行的恢复选择。

### Scenario S7: 需要人工介入

- Scenario id: `S7`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s7`
- Dimension: `environment_interruption`

Given
- 任务因登录、验证码、访问限制、文件选择或未知页面状态需要人工操作。

When
- App 通知用户并打开对应浏览器现场。

Then
- 用户可以接管、完成必要人工步骤，再选择恢复或终止任务。
- App 不要求用户通过技术 refs 找到正确现场，也不把人工介入误报为任务成功。

### Scenario S8: 授权范围不足

- Scenario id: `S8`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s8`
- Dimension: `security_permission`

Given
- 当前动作在 Lode 或 Harbor 的权威声明范围内，但超出当前全局或 task/operation scope grant。

When
- Core 返回需要用户决定的授权结果。

Then
- App 提供允许一次、当前 scope 允许或拒绝，并清楚展示目标和动作。
- 超出权威声明、目标绑定或有效期的动作直接拒绝，不提供扩大声明的入口。
- App 不建立站点专用审批状态机，也不保存授权 truth。

### Scenario S9: 按需查看来源与诊断

- Scenario id: `S9`
- Scenario locator: `.loom/stories/APP-308.md#scenario-s9`
- Dimension: `edge_case`

Given
- 用户或维护者需要核对结果来源或排查失败。

When
- 用户主动展开来源、运行详情或诊断。

Then
- App 可以展示 owner 提供的页面、时间、阶段、refs、trace 和原始错误。
- 关闭详情后，主要业务结果、失败原因和下一步仍保持清晰，不被诊断信息替代。
